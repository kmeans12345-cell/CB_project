프로젝트 개요
문명6 스타일 헥사곤 타일 기반 1인 전략 게임. 플레이어 vs AI. 순수 HTML/CSS/JS로 구성.
진입 파일: strategy_hex_v3.html

기술 스택

순수 HTML + CSS + JavaScript (프레임워크 없음)
SVG로 헥사곤 맵 직접 렌더링
외부 의존성: Google Fonts (Noto Serif KR, JetBrains Mono)


파일 구조

css/styles.css             — 전체 UI 스타일
js/01-core.js              — 헥스 좌표/거리, 이동 비용, 상수, 유닛 정의
js/02-state.js             — 덱 생성, 전역 상태, 공통 헬퍼
js/03-render-ui.js         — SVG 맵 렌더링, HUD, 손패/툴팁
js/04-obstacles.js         — 산/강 자동 랜덤 배치 단계
js/05-player-actions.js    — 선택, 배치, 이동, 공격, 건축물, 예약 이동
js/06-ai-turns-endgame.js  — AI 턴, 턴 종료, 승패, 초기화


맵 구조

10×10 헥사곤 그리드 (offset 좌표계, even-r, pointy-top)
좌표: q = 열, r = 행
플레이어 진영/킹 위치: (1, 8) / AI 진영/킹 위치: (8, 1)
시작 공병 위치: 플레이어 (1, 7) / AI (8, 2)
진영 구분 시각화는 제거됨. 배치는 시작 지점 반경만 사용
그리드 인덱스: ri(q, r) = r * COLS + q

핵심 헥스 함수
hexDist(q1,r1,q2,r2)     — 헥스 거리 (cube 좌표 변환)
hexNeighbours(q,r)        — 6방향 인접 헥스 반환
moveCost(cell)             — 강/다리 상태에 따른 기본 이동 비용 계산
moveCostAt(q,r,owner)      — 방패병 봉쇄 구역까지 반영한 실제 이동 비용 계산
hexReachable(q,r,steps)   — BFS 이동 가능 범위
bfsPathHex(sq,sr,tq,tr)   — 예약이동 전체 경로탐색
bfsNextStepHex(sq,sr,tq,tr) — 예약이동 다음 칸 반환

게임 흐름
setup
  → obs-random (산/강 자동 랜덤 배치)
  → battle (전투)
배치 단계 규칙
게임 시작 시 수동/교대 배치 없이 자동으로 장애물 생성
산 10개, 강 10개를 시작 지점 2칸 이내를 제외한 빈 칸에 무작위 배치
강은 산 인접/체인 연결 조건 없이 독립 배치
맵 중앙을 가르는 진영 구분선 및 아군/적군 절반 제한 제거

특수 규칙

킹: 각 진영 위치에 자동 배치, 방어 10, HP 200, 이동 불가. 킹이 쓰러지면 패배/승리
공병: 형평성을 위해 게임 시작 시 플레이어 (1, 7), AI (8, 2)에 각각 1기 자동 배치
창병 vs 기마병: +20 추가 피해
기사/기마병: 공격을 받을 때 30% 확률로 회피, 피해 무효
마법사: 인접 스플래시 20
포병: 인접 스플래시 10
방패병: 적이 방패병 인접 칸에 진입할 때 이동 비용 +1
궁병/포병/트레뷰셋/마법사: 공격자와 대상이 산을 가운데 두고 정반대에 있으면 공격 불가
트레뷰셋: 건축물에 2배 피해
공병: 건축물(벽/다리/감시탑) 건설 가능. 벽 HP 200, 다리 HP 200, 감시탑 HP 50, 1턴 건축물 1개 제한 (builtThisTurn 플래그)
벽: 빈 평지에만 설치 가능. 강/산/기물/다리/기존 벽이 있는 칸에는 설치 불가. 건설 시작 후 1턴 뒤 완공
공병 원거리 설치: 건축물 목표가 멀면 설치 가능한 인접 위치까지 이동 예약, 도착 후 다음 턴 종료에 자동 설치
다리: 강에만 설치 가능, 적/아군 모두 사용 가능, 설치된 강은 평지 취급. 건설 시작 후 3턴 뒤 완공
다리 공격: 트레뷰셋/포병만 가능. 다리 위 기물이 있으면 기물/다리 공격 선택
다리 파괴: 다리 위 기물이 있으면 HP와 무관하게 즉사
감시탑: 코스트 3, 건설 시작 후 2턴 동안 공병 이동 불가, 완공 시 탐색범위 5 제공
감시탑 주둔: 감시탑 칸에 유닛이 함께 있을 수 있으며, 공격 시 유닛/감시탑 중 대상을 선택
감시탑 파괴: 주둔 유닛보다 감시탑이 먼저 파괴되면 주둔 유닛은 최대 HP의 절반 피해
감시탑 점령: 주둔한 적이 없는 적 감시탑은 유닛이 진입해 점령 가능


덱 구성 (각 플레이어 동일)

기본 5종 × 각 3장 = 15장
특수 5종 × 각 1장 = 5장
총 20장, 시작 시 5장 드로우, 매 턴 1장 드로우


토큰 시스템

매 턴 기본 10개 지급
이동: 평지/다리 1토큰 / 강 진입: 2토큰 / 벽 설치: 2토큰 / 다리 설치: 2토큰
미사용 토큰 → 저금통 자동 저축 (남은 토큰 전부, 총 최대 20개)
이동 비용은 현재 턴 동작 토큰(tokens)에서 직접 차감되며, 저금통으로 대체 지불하지 않음
배치/건설 등 이동 외 비용은 저금통 코인부터 먼저 차감하고, 부족분을 현재 턴 토큰에서 차감


예약 이동

기물 이동은 즉시 이동하지 않고 예약으로만 처리
기물 최초 배치는 아군 킹 주변 6칸으로 제한
기물을 선택한 상태에서 한 번에 이동할 수 없는 위치를 클릭하면 자동으로 이동 예약
예약 순서는 playerReservationQueue / aiReservationQueue 배열로 보존
턴 종료 시 양 팀 예약을 같은 순번끼리 한 칸씩 순차 실행하며 선으로 경로 표시
예약 실행은 유닛별 턴당 이동 가능 칸/비용을 넘지 않음. 공병 설치예약도 동일 적용
예약 위치가 적 기물과 겹치면 해당 지점에서 한쪽이 죽을 때까지 충돌 전투
충돌 전투 중 대상 헥스가 점멸하는 전투 애니메이션 표시
강은 이동 비용 2, 다리가 설치된 강은 평지처럼 이동 비용 1
예약 이동 경로 탐색은 다리 없는 강을 우선적으로 우회하고, 다리가 있는 강은 우회하지 않음
산/기물/벽이 있는 칸은 예약 목표로 지정 불가


진영 체력 시스템

양 진영에는 HP 200의 킹이 자동 배치됨
킹 사망 → 즉시 게임 종료
기존 진영 HP 표시는 킹 HP 표시로 대체
applyBaseDamage() — endTurn() 내에서 AI 턴 전에 호출


전장의 안개

맵은 처음에 미탐색 상태로 어둡게 표시
플레이어 기물의 탐지 거리 안에 있는 헥스만 밝게 표시
탐지 거리는 각 유닛의 sight 스탯으로 별도 관리
근거리 유닛 탐지거리: 1 / 장거리 유닛 탐지거리: 2
한 번 밝혀진 헥스는 다시 시야 밖이 되어도 지형 정보는 어둡게 유지
시야 밖 적 기물과 건축물은 표시하지 않으며, 시야 밖 대상은 공격 불가

렌더링

SVG polygon으로 각 헥스 그림

refreshAll() — 전체 맵 갱신
유닛 테두리: 아군 = 초록(#3a9a3a), 적군 = 빨강(#e05050)
CSS 클래스: .unit-p / .unit-e (SVG style 태그 내 정의)
선택된 헥스: .sel, 이동 가능: .hl, 공격 대상: .atk


승리 조건

진영 HP 0 — 즉시 종료
40턴 판정 — 기물 수, 진영 HP, 플레이어 저금통, 감시탑 점령 수를 합산한 점수로 승리 판정


알려진 미완성/개선 여지

AI 로직이 단순 (그리디, 랜덤 배치). 강화 가능
강 배치 시 AI가 산이 없으면 강을 못 놓는 경우 발생 가능 → 예외 처리 추가 권장
트레뷰셋 vs 벽 공격 시 target 확인 로직 (grid[ri(tq,tr)].wall) — 유닛과 혼재 시 우선순위 확인 필요
AI 예약 이동은 아직 단순 목표 접근 중심이라 예약 충돌/경로 겹침 최적화 여지 있음


추가 기획안

1차 목표: 전술 선택지를 늘리되, 현재 순수 HTML/CSS/JS 구조와 예약 이동 중심 규칙은 유지한다.

우선순위 A — 규칙 완성도
- 벽/다리/감시탑 공격 우선순위 UI를 모달이 아닌 전용 선택 패널로 개선
- 예약 공격 유지 규칙 로그를 플레이어가 더 쉽게 읽도록 예약 목록에도 상태 표시
- 40턴 판정 점수 가중치 밸런싱: 기물, 진영 HP, 저금통, 감시탑 점령 수 비중 조정
- 방패병 봉쇄 구역 시각화: 적 이동 비용 +1 구역을 은은한 테두리로 표시

우선순위 B — 전략 확장
- 지휘관/버프 유닛 추가: 2칸 내 아군 공격 +5 또는 방어 +5를 제공하되 직접 공격력은 낮게 설정
- 정찰병 추가: 낮은 HP, 높은 이동력/탐지거리로 안개 탐색을 담당
- 공병 전문화: 벽/다리/감시탑 중 하나를 선택해 건설 속도 또는 비용 보너스를 주는 업그레이드형 규칙 검토
- 감시탑 점령 보상: 점령한 감시탑 수에 따라 매 턴 추가 시야 또는 토큰 1개 보너스 제공

우선순위 C — AI 개선
- AI 목표 점수화: 킹 접근, 저HP 적 공격, 감시탑 점령, 힐러 보호를 가중치로 계산
- AI 예약 충돌 회피: 아군 AI 기물이 같은 목표/경로를 과도하게 겹치지 않도록 예약 전 검사
- AI 건축 판단: 공병이 무작위가 아니라 병목 지점에는 벽, 강에는 다리, 전선에는 감시탑을 우선 건설
- AI 힐러 위치 선정: 회복 후 안전한 방향으로 후퇴하거나 킹 주변에 머무르는 행동 추가

우선순위 D — 사용성/표현
- 예약 목록 UI 개선: 이동/공격/건설 예약을 유형별 아이콘과 취소 버튼으로 표시
- 전투 로그 필터: 시스템/전투/플레이어/AI 로그를 토글로 분리
- 카드 툴팁 확장: 특수 규칙, 탐지거리, 예약 가능 여부를 카드에서 바로 확인
- 미니 튜토리얼: 첫 턴에 배치, 예약 이동, 안개, 건축물 설명을 한 번씩 안내

권장 구현 순서
1. 방패병 봉쇄 구역 시각화와 봉쇄 비용 밸런싱
2. 예약 공격 상태를 예약 목록 UI에 표시
3. AI 목표 점수화와 예약 충돌 회피
4. 정찰병/지휘관 등 신규 유닛 추가
5. 공격 대상 선택 패널과 로그 필터 개선

기획상 주의점
- 신규 유닛은 덱 총량을 크게 늘리기보다 특수 1장 카드로 시작해 밸런스를 확인한다.
- 시야 밖 정보가 드러나는 UI는 추가하지 않는다. 예약선, 경로, 테두리, 로그 모두 안개 규칙을 우선한다.
- AI 강화는 한 번에 완성하기보다 점수 함수만 먼저 도입해 기존 그리디 로직을 점진적으로 대체한다.
